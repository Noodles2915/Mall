# 商城系统接口清单(前后端对接用)

## 通用说明

- 基础路径：`/api`
- 鉴权方式：JWT
- 需要登录的接口，请在请求头携带：

```http
Authorization: Bearer <access_token>
```

- 统一返回结构：

```json
{
	"code": 0,
	"message": "ok",
	"data": {}
}
```

## 用户和地址系统

### 1. 注册

- 接口：`POST /api/auth/register/`

请求示例：

```json
{
	"username": "alice",
	"email": "alice@example.com",
	"password": "123456"
}
```

返回示例：

```json
{
	"code": 0,
	"message": "ok",
	"data": {
		"access": "<access_token>",
		"refresh": "<refresh_token>",
		"user": {
			"id": 1,
			"username": "alice",
			"email": "alice@example.com",
			"avatar": null
		}
	}
}
```


## 订单状态机流程

订单状态转换如下：

```
待支付 → 待发货 → 待收货 → 已完成
   ↓
 已取消
```

- **待支付**：订单刚创建，用户需要支付。用户可以取消此状态下的订单。
- **待发货**：用户已支付，等待商家发货。用户可以取消此状态下的订单。
- **待收货**：商家已发货，等待用户收货。用户可以确认收货。
- **已完成**：用户已确认收货，交易完成。
- **已取消**：订单被取消，库存已恢复。
