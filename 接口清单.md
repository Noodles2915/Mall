# 商城系统接口清单(前后端对接用)

## 通用说明

- 基础路径：`/api`
- 鉴权方式：JWT
- 需要登录的接口，请在请求头携带：

```http
Authorization: Bearer <access_token>
```

- 统一返回结构：

```json
{
	"code": 0,
	"message": "ok",
	"data": {}
}
```

## 用户和地址系统

- 接口：`POST /api/auth/register/`

### 资质申请相关（用户中心）

#### 1. 提交资质申请

- 接口：`POST /api/auth/qualification-applications/`（需登录）
- 说明：`application_type` 支持 `merchant`（商家资质）和 `staff`（工作人员资质）
- 限制：同一用户同一类型在“待审核”状态下只能存在一条申请

请求示例：

```json
{
	"application_type": "merchant",
	"reason": "主营3C数码，具备稳定供货渠道，联系电话13800138000"
}
```

返回示例：

```json
{
	"code": 0,
	"message": "ok",
	"data": {
		"id": 1,
		"application_type": "merchant",
		"application_type_display": "商家资质",
		"reason": "主营3C数码，具备稳定供货渠道，联系电话13800138000",
		"status": "pending",
		"status_display": "待审核",
		"review_note": "",
		"created_at": "2026-03-10T10:00:00+08:00",
		"reviewed_at": null
	}
}
```

#### 2. 我的资质申请列表

#### 3. 资质申请列表（管理员）

- 接口：`GET /api/auth/admin/qualification-applications/`（需登录且是管理员）
- 可选筛选参数：
- `status`：`pending/approved/rejected`
- `application_type`：`merchant/staff`
- `user_id`：按用户ID筛选

请求示例：

```http
GET /api/auth/admin/qualification-applications/?status=pending&application_type=merchant
Authorization: Bearer <admin_access_token>
```

返回示例：

```json
{
	"code": 0,
	"message": "ok",
	"data": [
		{
			"id": 1,
			"application_type": "merchant",
			"application_type_display": "商家资质",

			"username": "alice",
			"user_role": "normal",
			"reviewed_by_id": null,
			"reviewed_by_username": ""
		}
```

#### 4. 审核资质申请（管理员）

- 接口：`POST /api/auth/admin/qualification-applications/<pk>/review/`（需登录且是管理员）
- 说明：
- `status` 仅支持 `approved` 或 `rejected`
- 当 `status=rejected` 时，`review_note` 必填
- 商家资质申请审核通过后，系统会自动将用户角色更新为 `merchant`

请求示例：

```json
{
	"status": "approved",
	"review_note": "资料完整，审核通过"
}
```

返回示例：

```json
{
	"code": 0,
	"message": "ok",
	"data": {
		"id": 1,
		"application_type": "merchant",
		"application_type_display": "商家资质",
		"reason": "主营3C数码，具备稳定供货渠道，联系电话13800138000",
		"status": "approved",
		"status_display": "已通过",
		"review_note": "资料完整，审核通过",
		"created_at": "2026-03-10T10:00:00+08:00",
		"reviewed_at": "2026-03-10T11:00:00+08:00",
		"user_id": 2,
		"username": "alice",
		"user_role": "merchant",
		"reviewed_by_id": 1,
		"reviewed_by_username": "admin"
	}
}
```

## 订单状态机流程

订单状态转换如下：

```
待支付 → 待发货 → 待收货 → 已完成
   ↓
 已取消
```

- **待支付**：订单刚创建，用户需要支付。用户可以取消此状态下的订单。
- **待发货**：用户已支付，等待商家发货。用户可以取消此状态下的订单。
- **待收货**：商家已发货，等待用户收货。用户可以确认收货。
- **已完成**：用户已确认收货，交易完成。
- **已取消**：订单被取消，库存已恢复。
