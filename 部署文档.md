# 电商平台 - 部署说明书

## 项目概述

本项目是一个前后端分离的电商平台系统，包含用户认证、商品管理、购物车、订单系统等核心功能。

- **后端**：Django + DRF + MySQL
- **前端**：Vue 3
- **核心功能**：用户管理、商品展示、购物车、订单流程、后台管理

## 系统要求

- Python 3.10+
- Node.js 16+ 
- MySQL 5.7+ 或 MariaDB 10.3+
- pip (Python 包管理器)
- npm 或 yarn (前端包管理器)

## 后端部署步骤

### 1. 环境变量配置

在后端目录 `backend/` 下创建 `.env` 文件（或设置系统环境变量）：

```bash
# 数据库配置
DATABASE_PW=your_mysql_root_password
DATABASE_HOST=localhost
DATABASE_PORT=3306

# Django 秘钥
DJANGO_SKEY=your-secret-key-here

# 邮件配置（可选）
EMAIL_HOST=smtp.gmail.com
EMAIL_PORT=587
EMAIL_HOST_USER=your-email@gmail.com
EMAIL_HOST_PASSWORD=your-app-password
```

### 2. 创建虚拟环境

```bash
# 进入后端目录
cd backend

# 创建虚拟环境
python -m venv venv

# 激活虚拟环境
# Windows:
venv\Scripts\activate
# Mac/Linux:
source venv/bin/activate
```

### 3. 安装依赖

```bash
# 升级 pip
pip install --upgrade pip

# 安装项目依赖
pip install -r requirements.txt
```

### 4. 数据库迁移

```bash
# 生成迁移文件（如需要）
python manage.py makemigrations

# 应用迁移
python manage.py migrate
```

### 5. 创建超级管理员

```bash
python manage.py createsuperuser

# 按提示输入用户名、邮箱、密码
```

### 6. 导入初始数据（可选）

```bash
# 从 SQL 文件导入数据（如果有）
mysql -u root -p malldb < init_data.sql
```

### 7. 启动后端服务器

```bash
python manage.py runserver 0.0.0.0:8000
```

后端服务将在 `http://localhost:8000` 运行。

**管理后台**：`http://localhost:8000/admin/`

## 前端部署步骤

### 1. 进入前端目录

```bash
cd frontend/mall
```

### 2. 安装依赖

```bash
npm install
# 或
yarn install
```

### 3. 开发模式运行

```bash
npm run dev
# 或
yarn dev
```

前端应用通常在 `http://localhost:5173` 或 `http://localhost:3000` 运行。

### 4. 生产构建

```bash
npm run build
# 或
yarn build
```

构建后的文件在 `dist/` 目录下。

### 5. 预览生产构建

```bash
npm run preview
# 或
yarn preview
```

## API 配置

前端 HTTP 客户端默认连接到后端 API。修改 `frontend/mall/src/utils/http.ts` 中的 API 基础 URL：

```typescript
const API_BASE_URL = 'http://localhost:8000/api';
```

## 常见问题解决

### 数据库连接错误

**问题**：`No module named 'MySQLdb'` 或无法连接到 MySQL

**解决**：
```bash
pip install mysqlclient
# 确保 MySQL 服务运行中
# 检查 settings.py 中的数据库配置
```

### 跨域 (CORS) 错误

**问题**：前端请求后端 API 遭到 CORS 拒绝

**解决**：已在 `settings.py` 中配置远程。如需添加新的源，修改：

```python
CORS_ALLOWED_ORIGINS = [
    "http://localhost:3000",
    "http://localhost:5173",
    # 添加新的源
]
```

### Node modules 路径过长 (Windows)

**问题**：Windows 下路径超过 260 字符

**解决**：
```bash
# 启用 Windows 长路径支持
npm install --legacy-peer-deps
```

### 端口被占用

**问题**：8000 或 5173 端口已被使用

**解决**：
```bash
# 后端更改端口
python manage.py runserver 8001

# 前端在 package.json 中修改或命令行指定
npm run dev -- --port 3001
```

## 测试功能流程

1. **用户注册**：访问前端，注册新账户
2. **浏览商品**：在商品列表页面搜索和筛选
3. **加入购物车**：点击"添加到购物车"
4. **结算下单**：进入购物车，选择地址，提交订单
5. **支付**：模拟支付，订单变为"待发货"
6. **后台发货**：以超级管理员身份登录后台，发货
7. **确认收货**：用户确认收货，订单完成

## 后台管理

访问 `http://localhost:8000/admin/`，使用超级管理员账户登录。

### 主要管理功能

- **用户管理**：查看、编辑用户信息
- **商品管理**：添加、编辑、删除商品
- **订单管理**：查看订单详情，执行发货操作
- **购物车**：查看用户购物车内容
- **地址管理**：管理用户收货地址

## 数据库架构

### 核心表结构

- `user_user`：用户信息
- `user_address`：收货地址
- `product_productcategory`：商品分类
- `product_product`：商品信息
- `order_shoppingcart`：购物车
- `order_cartitem`：购物车项目
- `order_order`：订单
- `order_orderitem`：订单项目

## 性能优化建议

1. **数据库索引**：为频繁查询字段添加索引
2. **缓存策略**：使用 Redis 缓存热门商品
3. **前端优化**：启用 Gzip 压缩，使用 CDN
4. **API 分页**：大数据集使用分页

## 安全建议

1. **生产环境**：
   - 设置 `DEBUG = False`
   - 修改 `SECRET_KEY` 为强随机值
   - 使用 HTTPS
   - 配置 `ALLOWED_HOSTS`

2. **数据库**：
   - 使用强密码
   - 定期备份
   - 限制数据库访问 IP

3. **API 安全**：
   - 启用 CORS 白名单
   - 实现速率限制
   - 定期安全审计
